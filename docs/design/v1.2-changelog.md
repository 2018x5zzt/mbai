## C01: 归一化公式改为基于题库实际 min/max

### 问题

当前 `normalizeDimension(raw)` 使用硬编码 `(raw + 40) / 80`，假设理论范围 -40~+40。
实际题库中各维度可达范围差异巨大（control: -10~24, urgency: -1~5），导致归一化结果压缩在狭窄区间内。

### 变更

```ts
// 新增：启动时计算题库实际边界
function getDimensionBounds(questions: Question[]): Record<keyof Dimensions, { min: number; max: number }> {
  const bounds = {} as Record<keyof Dimensions, { min: number; max: number }>;
  DIM_KEYS.forEach((key) => {
    bounds[key] = questions.reduce(
      (acc, question) => {
        const values = question.options.map((o) => o.dimensions[key] ?? 0);
        return { min: acc.min + Math.min(...values), max: acc.max + Math.max(...values) };
      },
      { min: 0, max: 0 },
    );
  });
  return bounds;
}

// 替换原 normalizeDimension
function normalizeDimension(raw: number, key: keyof Dimensions, bounds: DimensionBounds): number {
  const { min, max } = bounds[key];
  if (max === min) return 0.5;
  return Math.max(0, Math.min(1, (raw - min) / (max - min)));
}
```

### 兼容性

- `dimension` 坐标体系不变（仍然是 [0, 1]）
- 坐标图展示不变
- 人格 `dimension` 字段含义不变

## C02: 彩蛋改为显式标签 + 确定性触发

### 问题

1. MARTYR: urgency < 0 选项仅 1 个（需要 5 个）→ 不可达
2. CULT: multiModel/emotion 归一化最大值分别仅 0.675/0.738 → 不可达
3. TURING: JURY 权重与 TROLL/CHECK 选项严重冲突 → 极难触发
4. 随机概率（0.25~0.40）导致同一答案多次测试结果不一致

### 变更

#### 新增标签

在 questions.json 相关选项中新增标签：

- `PATIENT`：标记温和/等待/不催促的选项（目标 >= 6 个选项）
- `RITUAL`：标记寻求象征解释/意义叙事的选项（给 ORACLE 用）

#### 彩蛋触发改为确定性

```ts
function calculateEasterEgg(
  answerIndexes: number[],
  dimensions: Dimensions,
  masterCode: string,
  bounds: DimensionBounds,
): EasterEgg | undefined {
  const tags = collectAllTags(answerIndexes);

  // MARTYR: 温和选项 >= 4 且没有 RAGE 标签
  const patientCount = tags.filter((t) => t === 'PATIENT').length;
  const rageCount = tags.filter((t) => t === 'RAGE').length;
  if (patientCount >= 4 && rageCount === 0) {
    return eggs.find((e) => e.code === 'MARTYR');
  }

  // CULT: 多模型 + 高情感 + 高验证
  const normalizedMulti = normalizeDimension(dimensions.multiModel, 'multiModel', bounds);
  const normalizedEmotion = normalizeDimension(dimensions.emotion, 'emotion', bounds);
  const checkCount = tags.filter((t) => t === 'CHECK').length;
  if (normalizedMulti >= 0.75 && normalizedEmotion >= 0.65 && checkCount >= 4) {
    return eggs.find((e) => e.code === 'CULT');
  }

  // TURING: JURY 主人格 + 质疑行为高频
  const trollCount = tags.filter((t) => t === 'TROLL').length;
  if (masterCode === 'JURY' && trollCount >= 3 && checkCount >= 4) {
    return eggs.find((e) => e.code === 'TURING');
  }

  return undefined;
}
```

#### 移除随机概率

删除 `createSeed` 和 `seededRandom` 函数。条件满足就触发，不再加概率门槛。

## C03: 新增 traitVector 多维匹配

### 问题

主人格判定仅靠 control/emotion 二维距离修正，无法区分 AGENT、POLY、JURY、DUMP 等核心差异在其他维度的人格。

### 变更

#### 数据结构新增

```ts
interface MasterPersonality {
  // ... 原有字段
  traitVector: Partial<Record<keyof Dimensions, number>>; // 新增
}
```

#### 每个人格的 traitVector

```json

VOID:   { "emotion": 0.9, "dependency": 0.7, "immersion": 0.5 }
POLY:   { "multiModel": 0.9, "control": 0.5 }
AGENT:  { "social": 0.9, "control": 0.6, "dependency": 0.5 }
JURY:   { "multiModel": 0.6, "control": 0.6, "entertainment": 0.4 }

CO:     { "entertainment": 0.7, "immersion": 0.5, "dependency": 0.4 }

PAPER:  { "control": 0.7, "dependency": 0.5 }

```

#### 评分公式

```ts
function calculateTraitSimilarity(
  userDimensions: Dimensions,
  traitVector: Partial<Record<keyof Dimensions, number>>,
  bounds: DimensionBounds,
): number {
  let similarity = 0;
  let count = 0;
  for (const [key, target] of Object.entries(traitVector)) {
    const normalized = normalizeDimension(userDimensions[key as keyof Dimensions], key as keyof Dimensions, bounds);
    similarity += 1 - Math.abs(normalized - target);
    count += 1;
  }
  return count > 0 ? similarity / count : 0;
}

// 最终评分
totalScore = masterWeights[code] * 1.0
           + traitSimilarity * 4.0
           - coordinateDistance * 2.0;
```
