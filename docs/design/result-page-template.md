# mbai 结果页与分享卡规范 v1.1

- 文档状态：开发就绪
- 文档角色：结果页结构、视觉规范、分享卡内容层级
- 当前目标：**结果页要像一张可截图传播的赛博人格档案，不像企业测试报告**

---

## 1. 结果页任务

结果页必须同时完成三件事：

1. **确认身份**：用户一眼看懂自己是谁
2. **形成传播**：用户想截图发出去
3. **顺手带走**：用户愿意复制提示词

---

## 2. 页面结构

当前前端实现不是简单纵向堆叠，而是：

> **顶部人格 Hero + 下方左右分栏的赛博档案页**

页面由 3 个大区块组成：

1. `Result Hero`
2. `主栏 Main Column`
3. `侧栏 Side Column`

### 2.1 当前实现结构

```ts
interface ResultPageV11 {
  hero: {
    brand: 'mbai｜赛博嘴脸图鉴';
    slug: string;
    title: string;
    verdict: string;
    deck: string;
    chips: string[];
    metrics: ['控制欲', '情感投入'];
    code: string;
    easterEgg?: string;
  };
  mainColumn: [
    '人格档案',
    '恶习叠加',
    '人格提示词'
  ];
  sideColumn: [
    '传播模块',
    '关系网络',
    '坐标与台阶',
    '怎么用它'
  ];
  sharePoster: {
    preview: '页面内预览版';
    export: '截图导出版';
  };
}
```

### 2.2 模块顺序

1. 顶部人格 Hero
2. 人格档案
3. 恶习叠加
4. 人格提示词
5. 传播模块
6. 关系网络
7. 坐标与台阶
8. 怎么用它

其中：

- `人格档案` 内部再拆成 `罪状 / 现场还原 / 赦免书`
- `坐标与台阶` 把坐标图和隐藏夸奖放在同一块，形成先打脸再给台阶的节奏
- `传播模块` 中同时存在页面内预览卡与单独导出卡

## 3. 首屏设计

首屏只负责两件事：

- 让用户认领结果
- 让用户产生截图冲动

### 3.1 首屏必须出现

- `mbai｜赛博嘴脸图鉴`
- 人格代号 `code`
- 中文封号 `title`
- 三字黑话 `slug`
- 一句话判词 `verdict`
- 简短 deck 文案
- 两个维度分数：`控制欲`、`情感投入`
- 两个副标签胶囊

### 3.2 首屏不建议出现

- 长段提示词
- 大段解释文本
- 太多按钮
- 完整坐标图说明文字

按钮操作当前已移入首屏下方的 `传播模块`，避免 Hero 直接挤满按钮。

---

## 4. 核心文案模块

### 4.1 一句话判词

这是结果页最重要的文案。

要求：

- 建议 `18~30` 个汉字
- 必须一眼打中
- 必须适合截图单独传播

### 4.2 罪状

作用：

- 把用户最不想承认但确实存在的习惯说破

要求：

- 建议 `40~70` 个汉字
- 要有攻击性，但不能羞辱到不敢发

### 4.3 现场还原

作用：

- 提供一个一看就有画面的典型场景

要求：

- 建议 `20~40` 个汉字
- 越具体越好
- 最好像聊天记录截图里的真实瞬间

### 4.4 赦免书

作用：

- 给用户一个台阶下
- 避免结果页只剩挨骂

要求：

- 建议 `20~40` 个汉字
- 真实优点，不要硬夸

---

## 5. 副标签区

副标签的作用是把“你是谁”进一步细化成“你还有什么毛病”。

### 5.1 展示规则

- 默认显示 `top 2`
- 顶部先给一条大号叠加结论
- 下方再拆开显示每个副标签的 `code + name + description`

### 5.2 合成描述写法

当前实现建议：

```text
叠加结论
你这次最明显的毛病是晒命狂：聊完第一反应就是截图发出去
```

这条结论文案建议在运行时按模板拼接，而不是对每个组合穷举写死。

---

## 6. 关系网络区

这是当前版本新增的差异化模块。

### 6.1 展示内容

- `宿敌人格`
- `人格 CP`

### 6.2 目的

- 提升结果页世界观感
- 增加转发时的二次讨论点
- 让不同人格之间形成可玩关系图谱

### 6.3 推荐呈现

```text
宿敌：DUMP｜代码乞丐
“我最烦那种复制报错就甩过来的人，需求描述都不会写”

人格 CP：CO｜共创搭子
“你其实缺一个能帮你把模糊需求翻译成结构化任务的人”
```

## 7. 赛博坐标图

### 7.1 用途

坐标图不是科学量表，而是视觉化的人格落点。

### 7.2 坐标定义

- `X 轴`：左侧是 `放任自流`，右侧是 `控制欲`
- `Y 轴`：下方是 `纯工具`，上方是 `当人了`

### 7.3 页面展示

- 浅灰色坐标轴
- 主人格参考点：黑色
- 用户实际点位：红色脉冲点
- 两个点同时显示，用于表达“你离这个人格原型有多近”

### 7.4 说明文案

不需要写长解释，只要一句：

`你在“控制欲”和“情感投入”坐标上，已经很难装了。`

---

## 8. 提示词区

### 8.1 展示策略

结果页展示 3 个版本：

- `轻量版`
- `情绪版`
- `效率版`

### 8.2 交互建议

- 默认展开 `轻量版`
- 其余 2 个版本折叠显示
- 当前版本统一复制当前激活版本

### 8.3 当前落地结构

- 左侧：大标题 + 版本切换
- 右侧：提示词正文
- 底部：复制按钮

### 8.4 写法原则

- 不要太工程化
- 要有明显人格味
- 用户复制后应能感到语气变化

## 9. 分享卡规范

### 9.1 第一版主规格

- 比例：`4:5`
- 建议尺寸：`1080 x 1350`

### 9.2 当前实现补充

当前实现使用两套卡：

- `preview`：放在结果页侧栏里，负责让用户当场看见“发出去是什么样”
- `export`：隐藏在页面外，用于 `html2canvas` 导出，避免污染实际布局
- 目标：适合朋友圈、小红书、群聊截图传播

### 9.2 分享卡内容层级

按重要性排序：

1. 人格代号 `code`
2. 中文封号 `title`
3. 三字黑话 `slug`
4. 一句话判词 `verdict`
5. 两个副标签
6. 品牌角标 `mbai`

### 9.3 第一版不放进分享卡的内容

- 长罪状
- 完整现场还原
- 完整提示词
- 宿敌 / CP 关系
- 详细坐标图说明

分享卡只负责传播，不负责讲完整故事。

## 10. 视觉规范

### 10.1 背景

- 纯白或米白
- 带极淡网格纹理
- 营造“坐标纸 / 档案纸”感觉

### 10.2 色彩

- 主色：黑色
- 强调色：红色
- 灰色用于坐标轴和弱信息层

### 10.3 字体建议

- 人格代号：超粗无衬线或钢印感字体
- 标题与判词：粗体无衬线
- 正文：常规无衬线

### 10.4 元素建议

- 人格代号用圆形徽章 / 钢印风
- 副标签用红色圆角胶囊
- 小标题用“罪状 / 现场还原 / 赦免书”这种档案式标签

### 10.5 整体气质

- 像一份认真得有点离谱的赛博人格档案
- 不像企业报告
- 不像严肃 MBTI 套壳
- 不像科技蓝产品页

## 11. CTRL 页面示例

```md
# CTRL｜赛博甲方
## 需求暴君

你不是在跟 AI 聊天，你是在给数字牛马开需求会。

【罪状】
你把 AI 当 24 小时待命的乙方，连“请”字都懒得打，结尾永远跟着“不对 / 重来 / 太水了”。

【现场还原】
复制一段需求，末尾加一句“你重新生成”，然后连发三次，中间还穿插“快点”。

【赦免书】
虽然你把 AI 当牛马，但你确实很擅长把“我觉得不太对”翻译成具体修改意见。

副标签叠加：连珠炮 × 晒命狂
= 需求还没说完就催稿，催完还要截图发群问“这 AI 是不是有点笨”

宿敌：DUMP｜代码乞丐
人格 CP：CO｜共创搭子

隐藏夸奖：你其实很擅长把模糊问题压成清晰指令。
```

## 12. 截图实现建议

### 12.1 截图区域

建议单独设计 `data-screenshot` 区域，内容仅保留：

- 品牌角标
- 人格主标题
- 判词
- 两个副标签
- 轻量视觉元素

### 12.2 导出方案

- 使用 `html2canvas`
- 导出 `png`
- 默认适配移动端宽度

### 12.3 按钮建议

结果页底部保留 3 个动作：

- `截图保存`
- `复制文案`
- `再测一次`

---

## 13. 设计验收标准

结果页与分享卡上线前至少要满足：

1. 不看说明，用户也能一眼知道自己是什么人格
2. 分享卡脱离长文仍然成立
3. 文案和视觉组合后不会像普通 MBTI 模板
4. 同时放 3 张不同人格卡时，差异足够明显
5. 首屏截图就有传播价值
