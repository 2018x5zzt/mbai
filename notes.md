## 3. 彩蛋触发可达性

### MARTYR（AI 殉道者）— 完全不可达

- 条件: patientCount >= 5（urgency < 0 的选项数量）
- 全题库仅 q1a 有 urgency = -1，最大 patientCount = 1
- 差距: 缺 4 个 urgency < 0 的选项

### CULT（赛博 Cult 领袖）— 完全不可达

- 条件: normalizedMulti > 0.8 且 normalizedEmotion > 0.8
- multiModel 最大原始值 = 14 → 归一化 = 0.675（需要 > 0.8）
- emotion 最大原始值 = 19 → 归一化 = 0.738（需要 > 0.8）
- 两个维度都达不到阈值

### TURING（图灵测试员）— 极难触发

- 条件: JURY 主人格 且 TROLL+CHECK >= 7
- 可贡献 TROLL/CHECK 的题: 8 题（理论可达 >= 7）
- 但 JURY 权重来源与 TROLL/CHECK 选项存在严重冲突
- q4d 给 JURY+3 但标签是 SHOW（不含 CHECK/TROLL）
- q14c 给 JURY+2 但标签是 SHOW（不含 CHECK）
- 要满足 trollCheck >= 7 就必须牺牲 JURY 权重，导致 JURY 难以胜出

## 4. 弱人格识别

| 人格    | 净入口 | 问题                            |

| ORACLE | 0      | 死人格：零权重入口，永远不会被选中 |
| MONO   | 1      | 近乎死亡：正权仅 +4，负权 -3    |
| CO     | 6      | 被 LULZ 压制：共享选项中 LULZ 权重更高 |

## 5. 核心结论

1. **ORACLE 是死人格**：永远不会被算出来
2. **MONO 近乎死亡**：净入口仅 1
3. **三个彩蛋全部不可达或极难触发**
4. **CO 被 LULZ 压制**：所有共享来源中 LULZ 权重都更高
5. **urgency 和 expression 维度无区分度**：跨度分别仅 0.075 和 0.038
